# PRD — B2B‑бот для агентств и отелей (VoC + отзывы)

Связанная спецификация: `SPEC_B2B_bot_agencies_hotels.md`

## 1. Обзор

**Название (рабочее):** B2B‑бот для агентств и отелей — VoC‑агент для турагентств (Telegram)  
**Форм‑фактор:** Telegram‑бот + веб‑форма загрузки файлов (минимум)  
**Цель (6 месяцев):** ≥ 500 000 ₽/мес выручки, 1–2 проекта параллельно

Бот помогает руководителю турагентства:

- автоматически собирать и анализировать диалоги из каналов продаж/поддержки;
- выделять **корневые проблемы сервиса** и “точки потери” клиента;
- обновлять **библиотеку шаблонов ответов** под типовые конфликтные ситуации;
- автоматически собирать метрики репутации (рейтинг/кол‑во отзывов) с **Яндекс / 2ГИС / Google**.

## 2. Для кого

**Основной пользователь:** директор операционного блока / директор агентства / владелец.  
**Вторичный пользователь:** старший менеджер, который поддерживает “банк шаблонов”.

**Тип компании:** небольшое турагентство, 1–20 менеджеров, коммуникации через WhatsApp/Telegram/Email/CRM/корпоративные чаты.

## 3. Проблема клиента

- нет системного понимания, почему клиент недоволен (голос клиента “размазан” по каналам);
- мало отзывов и слабая управляемость репутацией;
- нет единого стандарта ответов (разные менеджеры отвечают по‑разному);
- конфликты и возвраты “съедают” время и конверсию повторных продаж.

## 4. Повторяющийся сценарий

- **Еженедельно:** загрузить экспорт переписок → получить отчёт (топ проблем + рекомендации).
- **Ежедневно:** менеджер открывает банк шаблонов → копирует подходящий ответ → адаптирует и отправляет клиенту.
- **Еженедельно/ежемесячно:** мониторинг рейтинга и числа отзывов на Яндекс/2ГИС/Google.

## 5. Что входит в MVP

### 5.1 Импорт данных (без интеграций на старте)

- **WhatsApp:** импорт `.txt` (Export Chat)
- **Telegram:** импорт `JSON` (Export Telegram Data)
- **CRM / самописная CRM / Max / Email / корпоративные чаты (напр. Webim):** импорт `.csv` / `.xlsx` по согласованной схеме

### 5.2 Аналитика (LLM)

- кластеризация обращений по темам;
- выявление корневых проблем (топ‑10);
- “моменты потери” (где эскалирует конфликт);
- рейтинг менеджеров по риску жалоб (по стилю и теме, аккуратно);
- рекомендации: какие процессы/политики/коммуникации нужно улучшить.

### 5.3 Банк шаблонов ответов

- структура: **продукт** (авиа/отель/трансфер/тур) × **тип запроса** (возврат/обмен/перенос/документы/претензия/прочее);
- генерация новых шаблонов из выявленных проблем;
- поиск/фильтр + “копировать”.

### 5.4 Репутация (автоматически)

- сбор: рейтинг, число отзывов, последние N отзывов (если доступно) для 3 площадок;
- вывод динамики.

## 6. Чего не делаем в MVP

- бот сам не отправляет сообщения клиентам (только черновики/копирование);
- нет глубокой интеграции с amoCRM/Bitrix24/TravelLine/самописными CRM/Webim (только импорт);
- нет хранения “сырого” текста переписок после обработки (по умолчанию удаляем).

## 7. Монетизация и тарификация (черновик)

**Freemium (пробный):**

- импорт до ~100 диалогов/месяц
- отчёт за текущий месяц + бесплатная динамика 3 месяца (агрегаты)
- базовый банк шаблонов

**Pro:**

- 1 000 / 2 000 / 3 000 диалогов в месяц (ступени)
- динамика 12+ месяцев (агрегаты)
- расширенный рейтинг менеджеров и рекомендации

**Ценообразование (ориентир):** 15–49k ₽/мес в зависимости от объёма и числа менеджеров.

## 8. Метрики успеха

- время ответа менеджера (до/после) — опционально
- доля конфликтных кейсов (по классификации)
- рост повторных покупок (по данным агентства)
- рост рейтинга и кол‑ва отзывов на площадках
- MRR и churn

## 9. Риски и ограничения

- данные с отзывов могут требовать официальных API/ключей; где нельзя — используем безопасные альтернативы (агрегаты/ручной ввод/официальные инструменты).
- приватность переписок: анонимизация, минимизация, удаление.
- качество LLM: нужен контроль и “объяснимость” выводов.
