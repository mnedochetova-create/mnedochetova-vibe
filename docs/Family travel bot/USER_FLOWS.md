## User Flows — Family travel bot (Telegram)

Цель: описать пользовательские сценарии по этапам, чтобы использовать как основу для разработки (стейты, кнопки, события аналитики, тест‑план).  
Источник: `PRODUCT_CARD.md`, `TASK_2_Validation_Check.md`, `MARKET_RESEARCH_COMPETITORS.md`.

### Легенда
- **P0/P1/P2** — приоритет (как в `PRODUCT_CARD.md`).
- **Value moment** — первый «вау‑момент», который подтверждает ценность.
- **События** — будущие события аналитики (черновик, уточним в разработке).

### Tone of voice
- **База:** дружелюбный, вдохновляющий тон: тёплый, уважительный, без снисхождения; короткие абзацы; конкретика вместо общих лозунгов.
- **Запреты:** агрессивное давление, стыд, обесценивание выбора пользователя, «шантаж скидкой»; формулировки, которые подрывают доверие к источникам и партнёрским ссылкам.
- **Побуждение к действию (бронь, watchlist, следующий шаг):** через ценность и ясность («что станет легче / что вы экономите по времени и тревоге»), а не через манипуляцию.
- **Деловая лаконичность (ориентир):** в блоках с цифрами, сравнениями и алертами — кратко, по фактам, с контекстом (принцип сильных деловых материалов уровня Forbes и аналогов), без воды и без сенсационных заголовков.

Ниже — **единая методология текстов и диалогов** для приветствия, подсказок, карточек, сравнений и CTA на всех этапах. Она не воспроизводит книги дословно, а задаёт **структуру и критерии качества**, согласованные с идеями перечисленных авторов.

#### Каркас сообщения (гибко по контексту)
1. **Ясная рамка / позиция** — о чём речь и зачем это человеку (*Райс и Траут — позиционирование; Огилви — одна сильная мысль; Котлер — ценность на шаге воронки*).
2. **Факты и выгода простым языком** — что получит пользователь и на каких условиях (*Огилви, Котлер; Патель — предсказуемая структура и читабельность*).
3. **Доверие** — источники, дисклеймеры, «почему мы так говорим» (*Годин — честность и permission; Линдстром — согласованность опыта; Кавасаки — enchantment и предсказуемость*).
4. **Выбор без давления** — варианты A/B/C, право отказа, мягкий CTA (*Кэмп — снижение ощущения принуждения; Фишер и Юри — интересы и критерии вместо борьбы позиций*).
5. **Социальное доказательство и нормы** — этично и по делу (*Чалдини; Гладуелл — короткий узнаваемый пример*).
6. **Следующий шаг** — один понятный микродействие (*Вэйнчук — контекст и уместность; Годин — «что будет дальше»*).

## 1) Онбординг

**Обязательное правило:** любой первый контакт (личный чат `/start`, первое добавление в группу, первый запуск Mini App) начинается с **ONB-0 — приветственное сообщение**. Без него не переходим к сбору брифа и остальным шагам онбординга.

|----|----------|----------|--------------------------|------------------|---------------------|-------------|----------|--------------------|
| **ONB-0** | **Обязательное приветствие** | все | `/start`, первый контакт в личке, первое добавление в группу, первый запуск Mini App | **Дружелюбное, вдохновляющее приветствие** по **каркасу** и **Tone of voice**: кто бот → чем полезен → чем отличается (коротко) → 1–3 мягких CTA → честность про цены/источники; без давления на покупку | Понимание ценности, доверие к первому шагу | «хочется продолжить — понятно, зачем» | **P0** | `welcome_shown`, `onboarding_started` |

## 7) Решения по открытым вопросам

### Условия тарифа в карточке (MVP)

**Источник данных:** Travelpayouts Data API (кэш цен Aviasales), не полный GDS‑тариф.

- **Показываем из API (если поле есть в выбранном методе):** цена + валюта; маршрут (IATA → подписи городов у нас); даты вылета/возврата; число пересадок; класс (`trip_class`); при наличии — `found_at`, `expires_at`, авиакомпания/рейс (`cheap`/`direct`), `gate`.
- **Не выводим как факт из API:** багаж, возврат/обмен, сборы — только напоминание, что **итоговые условия тарифа** на сайте партнёра после перехода по ссылке.
- Подробная таблица полей и эндпоинтов: `TRAVELPAYOUTS_AVIASALES_FIELDS.md`.

**Связка с API в репозитории:** токен только в `bot/.env` (не в git); шаблон переменных — `bot/.env.example`. Ключ из чата/тикетов считать скомпрометированным и перевыпустить в кабинете Travelpayouts.

### ROI / выгода в ₽ в алерте

- **MVP:** эвристика — разница между **текущей отметкой в кэше** и **базовой** (цена при создании watchlist или минимум за N дней, если храним); плюс текстовая оценка риска «цена может измениться до оплаты».
- **P1:** история замеров (свои сохранённые точки), если появится стабильное хранение.

### Режим по умолчанию в группе

- **Дефолт:** тихий режим — ответы по **@упоминанию** бота; активный (по ключевым словам) — только после явного выбора, чтобы не спамить семейный чат.

### Прокси «покупки» (раз мы не видим оплату)

- Клик по партнёрской ссылке / `partner_click`; открытие deeplink; опционально постбек/affiliate‑отчёты Travelpayouts, если доступны для вашей интеграции.
