## Task 1 — Idea Filter (scoring)

Источник методики: `docs/ideas/Ideas Shaping Practice Materials.md`

Правила:

- **Passion**: 1–5 (1 = meh, 5 = obsessed)
- **Expertise**: 1–5 (1 = learning curve, 5 = domain pro / unfair advantage)
- **10‑Year Test**: если **No** → идею откладываем и **не учитываем в приоритизации**
- **Total** = Passion + Expertise (максимум 10)

Примечание по колонке API:

- **Да** — есть прикладной доступ к ключевым данным/функциям через API на старте MVP.
- **Частично** — API доступен только для части сценария, либо сильно ограничен (официальные интеграции/доступы/ручной ввод).
- **Нет** — ядро сценария не зависит от внешнего API или готового API-доступа нет.
